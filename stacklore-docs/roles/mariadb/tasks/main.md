# mariadb · tasks

## What It Does
Installs MariaDB server, secures the default installation by removing anonymous users and the test database, sets a root password, creates an application database and user, and optionally configures the server to accept remote connections.

## When To Use It
Run this role on any host that needs to act as a MariaDB server — typically during initial provisioning or when setting up a new environment. This handles the full bootstrap: installation, security hardening, and application database/user setup in one shot.

## Requirements
- Debian/Ubuntu-based OS (uses `apt`)
- `python3-pymysql` package (installed as part of this role, needed for Ansible's `mysql_*` modules)

## Variables

| Variable | Default | Description |
|---|---|---|
| `mariadb_root_password` | *required* | Root user password. Must be set before running this role. |
| `mariadb_db` | *required* | Name of the application database to create. |
| `mariadb_user` | *required* | Name of the application database user. |
| `mariadb_password` | *required* | Password for the application database user. |

## What It Changes On The Host
- Installs `mariadb-server`, `mariadb-client`, and `python3-pymysql` packages
- Starts the MariaDB service and enables it for boot
- Sets MariaDB root password and enforces `mysql_native_password` plugin
- Removes all anonymous user accounts
- Deletes the default `test` database
- Creates the application database with UTF-8MB4 encoding
- Creates the application user with full privileges on the application database
- Modifies `/etc/mysql/mariadb.conf.d/50-server.cnf` to bind to `0.0.0.0` (if the "Allow remote connections" task runs)
- Restarts MariaDB service (if config changes are made)

## Example Usage

```yaml
- hosts: db_servers
  roles:
    - role: mariadb
      vars:
        mariadb_root_password: "{{ vault_mariadb_root_password }}"
        mariadb_db: "myapp_db"
        mariadb_user: "myapp_user"
        mariadb_password: "{{ vault_mariadb_app_password }}"
```

## Gotchas
- **Remote connections task is commented/disabled by design.** The last task modifies the bind address but doesn't include a `when` condition. If you uncomment it or leave it enabled, it will bind MariaDB to all interfaces — only do this in dev/test environments, never in production without firewall rules in place.
- **Root password is idempotent only on first run.** The initial "Set MariaDB root password" task uses `check_implicit_admin: true` which allows socket auth without a password. On subsequent runs, this task will skip but subsequent tasks will still need the correct password in `mariadb_root_password` or they'll fail.
- **All four variables are required.** If any of `mariadb_root_password`, `mariadb_db`, `mariadb_user`, or `mariadb_password` are undefined, tasks will fail.
- **Waits only 30 seconds for port 3306.** On slow or heavily loaded systems, this timeout may be too short, causing the role to fail. Increase if needed.
